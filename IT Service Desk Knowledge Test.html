Section 1: Technical Knowledge (15 Questions)

1. What does DHCP stand for and what is its primary function?
DHCP stands for Dynamic Host Configuration Protocol. Its primary function is to automatically assign IP addresses to devices on a network, simplifying network administration.
2. Which of these is a private IP address range?
192.168.1.1 is part of the private IP address range (192.168.0.0 to 192.168.255.255). Private IP addresses are used for local area networks and are not routable on the public internet.
3. A user cannot print. What is the FIRST step you should take?
Always start with the simplest physical checks before moving to software/driver troubleshooting. This is "checking the basics" and often resolves the issue quickly.
4. What is the primary purpose of DNS?
DNS acts like a phonebook for the internet, converting human-readable names (google.com) to machine-readable IP addresses (142.251.42.206).
5. In Windows, which tool would you use to check if a network cable is functioning properly on a physical level?
In Windows 10/11, the icon in the system tray would show a red X or warning if the cable is unplugged or faulty. `ping` tests logical connectivity, not the physical cable itself.
6. What does the "A" in the ITIL service lifecycle term "BYOD" typically stand for?
BYOD stands for "Bring Your Own Device." This question checks attention to detail and familiarity with common IT acronyms.
7. A blue screen of death (BSOD) on a Windows machine typically indicates:
A BSOD is a stop error caused by critical issues like failing hardware or corrupt drivers. It requires immediate attention and troubleshooting.
8. Which command is used to release and renew a DHCP-assigned IP address from the command prompt?
This clears the current IP address and requests a new one from the DHCP server. It's a common troubleshooting step for network connectivity issues.
9. What is the main function of a router in a network?
A switch manages traffic within a LAN (using MAC addresses), while a router connects LANs to WANs (like the internet) using IP addresses.
10. What does VPN stand for?
A VPN extends a private network across a public network, allowing for secure remote access to company resources.
11. Which port is typically used for secure HTTP (HTTPS) traffic?
Port 80 is for unencrypted HTTP. Port 443 is the standard port for HTTPS traffic, which uses SSL/TLS encryption.
12. In Active Directory, what is a "OU" used for?
An Organizational Unit (OU) is a container in Active Directory used to organize objects and apply Group Policies for management and security purposes.
13. What is the purpose of the `ping` command?
`ping` sends ICMP packets to a destination and reports if it gets a reply and how long it took. It's a fundamental network troubleshooting tool.
14. A user reports their computer is running very slowly. What is a likely NON-malware related cause?
While malware is a common cause, other reasons include too many programs running, a needed reboot, or a resource-intensive process like a Windows update.
15. RAID 1 is primarily known for:
RAID 1 writes identical data to two disks simultaneously, creating a mirror. If one disk fails, the other has a perfect copy, ensuring data redundancy.

Section 2: Troubleshooting & Methodology (10 Questions)

16. What is the first step of the general troubleshooting methodology?
You must fully understand what the problem is before you can attempt to solve it. This involves gathering information and asking clarifying questions.
17. A user calls and says, "My internet is down." What is the BEST follow-up question?
Gathering specific details is crucial for effective troubleshooting. "My internet is down" is vague; it could be the browser, one website, WiFi, etc.
18. After implementing a solution to a problem, what should you do next?
You must ensure your fix actually worked and didn't break anything else. Documenting (B) is part of this final step but verification comes first.
19. You've solved a user's issue. What is the best way to end the call?
This is a professional and customer-focused way to conclude an interaction, ensuring all the user's needs are met before ending the call.
20. What does "escalating a ticket" mean?
Escalation happens when the issue is beyond the current support level's scope or expertise. It ensures the ticket gets to the right people for resolution.
21. When remote troubleshooting, which tool is most commonly used to securely take control of a user's desktop?
These tools allow an analyst to see and control the user's desktop remotely to diagnose and fix issues directly, providing efficient support.
22. Why is it important to document the steps taken during troubleshooting?
Good documentation saves time and builds a valuable resource for the entire team. It helps in resolving similar issues faster in the future.
23. A user is frustrated and speaking loudly. What is the best approach?
De-escalation is a key skill. Matching their frustration will only make the situation worse. Staying calm and empathetic helps resolve the issue effectively.
24. What is "Active Listening" in a service desk context?
Active listening is about engaged listening, not just hearing. It involves fully concentrating on what is being said rather than just passively hearing the message.
25. When dealing with a major outage affecting multiple users, what is the priority?
For major incidents, communication and restoration are paramount. Individual ticket logging is secondary to resolving the widespread issue.

Section 3: Customer Service & Soft Skills (10 Questions)

26. What is the most important trait of a successful Service Desk Analyst?
Technical skills can be taught, but a strong customer-focused attitude is fundamental for this role. Service desk is as much about people as it is about technology.
27. A user is explaining a problem you've heard 10 times today. You should:
Every user deserves respect and your full attention. Making them feel heard is an important part of providing good customer service.
28. When explaining a technical solution to a non-technical user, you should:
The goal is to be understood, not to impress. Using clear, non-technical language helps the user learn and potentially solve similar issues themselves in the future.
29. What does SLA stand for?
An SLA defines the expected level of service between a provider and a customer (e.g., response and resolution times). It's a key component of IT service management.
30. A caller asks for help with an application you are not familiar with. You should:
Honesty and a willingness to find a solution are better than providing incorrect information. This approach builds trust with the user.
31. Effective communication over the phone relies heavily on:
Without visual cues, your voice is your primary tool for communication and building rapport. A friendly, clear tone can significantly improve the customer experience.
32. Why is it important to set realistic expectations with a user?
If you say "5 minutes" but it will take an hour, the user will become frustrated. Being honest about timelines manages expectations and avoids dissatisfaction.
33. What should you do if you make a mistake while helping a user?
Taking ownership builds trust and shows professionalism. Everyone makes mistakes; how you handle them defines your professionalism.
34. The phrase "I understand your frustration" is an example of:
It shows the user you recognize and acknowledge their feelings, which helps build rapport and de-escalate tense situations.
35. Before transferring a call, you should always:
This is courteous and prevents the user from feeling dismissed or having to repeat their story from the beginning to the next agent.

Section 4: Processes & Security (5 Questions)

36. What is the primary goal of Incident Management?
Incident Management is reactive. Problem Management is focused on finding the root cause. The immediate goal is restoration of service.
37. A user calls asking for another user's password. What should you do?
Sharing passwords is a critical security violation. Always follow the official, secure process for password resets.
38. What is a "phishing" email?
Phishing emails often impersonate legitimate companies to trick users into giving up passwords or financial info. They're a common security threat.
39. Why should you lock your workstation when you step away from your desk?
This is a fundamental security practice known as "clean desk" or "workstation locking" policy. It prevents unauthorized access while you're away.
40. What is the purpose of a Knowledge Base?
A knowledge base empowers both analysts and users to find solutions quickly, improving efficiency and reducing resolution times for common issues.
